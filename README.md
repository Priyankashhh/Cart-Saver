# 🛡️ CartSaver

> **AI-Powered Customer Retention & E-Commerce Automation Platform**
> Optimize shopping cart recovery, analyze user drop-offs, and automate personalized marketing communications using NVIDIA NIM and real-time dashboard analytics.

---

## 🚀 Project Overview

**CartSaver** is a customer retention platform that integrates transactional checkout funnels with LLM intelligence and multi-channel notification engines. By tracking user behavior from cart creation to checkout success, CartSaver identifies drop-off points, segments users by purchase intent, and leverages NVIDIA NIM (Llama 3.1 70B Instruct) to generate highly personalized conversion incentives delivered via Email, SMS, and WhatsApp.

---

## 🛠️ Technology Stack

- **Backend Logic**: Python 3.13+
- **Database**: SQLite (Zero-config default) & PostgreSQL support
- **AI Core**: NVIDIA NIM (Meta Llama 3.1 70B Instruct API)
- **Notifications**: SendGrid API (Email), Twilio API (SMS & WhatsApp)
- **Scheduling**: APScheduler (Advanced Python Scheduler)
- **Frontend Dashboard**: HTML5, Vanilla CSS3 (glassmorphic dark mode), JavaScript (ES6), ApexCharts, Lucide Icons

## ⚙️ Setup and Installation

### 1. Prerequisites
- Python 3.10+ installed
- (Optional) PostgreSQL database if scaling beyond SQLite

### 2. Installation
Clone the repository and install the dependencies:
```bash
git clone https://github.com/Priyankashhh/Cart-Saver.git
cd Cart-Saver
pip install -r requirements.txt
```

### 3. Environment Variables
Copy `.env.example` to `.env` and populate your credentials:
```bash
cp .env.example .env
```
Key configuration settings in `.env`:
* **`DB_BACKEND`**: Choose `sqlite` (default) or `postgresql`.
* **`NVIDIA_API_KEY`**: Your NVIDIA NIM API key for generating AI recovery briefings.
* **`SENDGRID_API_KEY`**: SendGrid API key for email recovery campaigns.
* **`TWILIO_ACCOUNT_SID` & `TWILIO_AUTH_TOKEN`**: For SMS and WhatsApp notification delivery.

## 💻 CLI Usage

The core retention pipeline is managed via the command-line interface.

| Command | Action |
|---|---|
| `python -m cartsaver.main --setup` | Creates database tables and seeds mock user journey data. |
| `python -m cartsaver.main --run-now` | Runs the retention pipeline, calculates incentives, and prepares notifications. |
| `python -m cartsaver.main --simulate` | Simulates conversions and responses to previous recovery campaigns. |
| `python -m cartsaver.main --export` | Exports compiled user segments and funnel metrics to CSV files. |
| `python -m cartsaver.main --start-scheduler` | Starts a persistent APScheduler process (defaults to daily at 9:00 AM). |
| `python -m cartsaver.main --all` | Sequentially runs Setup ➔ Pipeline ➔ Simulation ➔ CSV Export. |

## 📊 Interactive Analytical Dashboard

CartSaver includes a responsive analytical dashboard that reads metrics directly from the database and features an automated executive summary generated by Llama 3.1.

### How to Run:
```bash
python dashboard/run_dashboard.py
```
This command:
1. Spins up a local HTTP server at `http://localhost:8000`.
2. Automatically opens your default web browser to the dashboard index page.
3. Provides an interactive **"Refresh Data"** button that runs the database aggregation and regenerates Llama briefings on-demand.

### Visual Metrics Included:
- **KPI Cards**: Total Revenue, Checkout Conversion Rate (CVR), Revenue At Risk, and Recovery Potential.
- **Checkout Funnel**: Visualizes user drop-offs from *Product View* to *Add to Cart* to *Checkout Started* to *Purchase*.
- **Business Retention Health**: Gauge chart assessing current customer lifetime health.
- **Top Cart-Abandoned Products**: Spotlights specific products causing the highest drop-offs.
- **Campaign Channel Efficiencies**: Evaluates the cost-to-benefit ratio of Email vs. SMS vs. WhatsApp channels.

## 📈 Power BI Integration

Running `python -m cartsaver.main --export` outputs analytical files into the `exports_output/` folder. These files are optimized for loading into Business Intelligence platforms (like Power BI, Tableau, or Excel):
- **`funnel_stages.csv`**: Aggregated count per checkout funnel level.
- **`funnel_dropoffs.csv`**: Quantifies where customers leave the application.
- **`user_segments.csv`**: Detailed user profiles and cohort labels.
